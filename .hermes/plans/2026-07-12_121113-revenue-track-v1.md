# TimeLeak Revenue Track V1 Plan

> **For Hermes:** Implement this plan task-by-task after explicit user approval.

**Goal:** Ship the shortest truthful path from a cold visitor to one useful tomorrow repair, followed by a live $9.99 30-day-pass purchase.

**Architecture:** A single linear onboarding sends one complete 24-hour input to the four-stage agent workflow. Convex stores the canonical user, analysis, event, access, payment, and agent-run records; Cloudflare serves the product and server-side integrations; Dodo handles checkout and idempotent webhooks.

**Tech Stack:** Hermes with OpenAI, Convex, Cloudflare, Dodo Payments; ElevenLabs and LinkUp remain optional after the core revenue path works.

---

## 1. Locked user journey

1. Cold visitor lands on one promise and one CTA: map tomorrow and reclaim one honest block of time.
2. User enters email.
3. User enters sleep time and wake time.
4. User supplies tomorrow’s full-day schedule using the fastest initial V1 method; paste/manual text is sufficient for first value. Demo, screenshot, and short voice are allowed input modes but must not delay the core path.
5. User enters exactly one postponed life priority.
6. The system asks no follow-up unless the 24-hour map has a material gap or contradiction.
7. Day Mapper normalizes the full day and marks uncertainty.
8. Leak Detective selects exactly one defensible leak: time neither consciously chosen nor valued afterward.
9. Day Architect creates exactly one conflict-free repair for tomorrow without reducing sleep, essential work, caregiving, meals, or intentional rest.
10. Truth Judge validates arithmetic, privacy, feasibility, and product rules. On failure, revise once; on a second failure, return an honest limited result.
11. The user receives one complete free repair and can copy or download it.
12. Immediately after value, the user sees one paid offer: a one-time $9.99 30-day pass, immediate access, and a seven-calendar-day full-refund window.
13. Dodo completes checkout; an idempotent webhook updates payment and access in Convex.
14. Refresh and back navigation restore the existing result rather than creating duplicate analyses or purchases.

## 2. Minimum data model

- `users`: `email`, timestamps.
- `analyses`: `userId`, raw onboarding input (`sleepTime`, `wakeTime`, `schedule`, `priority`), validated analysis output, overall status, created/updated timestamps.
- `agentRuns`: `analysisId`, ordered stage (`day_mapper`, `leak_detective`, `day_architect`, `truth_judge`), status, latency, safe error, attempt number, timestamps.
- `events`: `userId`, optional `analysisId`, event name (including first-use/value events), timestamp, minimal metadata.
- `payments`: `userId`, Dodo checkout/payment identifiers, amount and currency, payment status, webhook event identifier for idempotency, timestamps.
- `access`: `userId`, entitlement type (`free_repair` or `thirty_day_pass`), start/end dates, status, payment reference, refund eligibility/deadline.

The production analysis output must match `contracts/timeleak-analysis.schema.json` exactly. That contract file does not currently exist in the repository and must be defined before implementation.

## 3. First four milestones

1. **Contract and truth engine:** Define the exact JSON contract and deterministic validation rules for one 24-hour map, one leak, one repair, protected-time constraints, arithmetic, uncertainty, and limited-result fallback.
2. **Under-two-minute free value:** Build the single-page onboarding and four-stage analysis path using the fastest schedule input, then support copy/download and resilient loading/error states.
3. **Revenue conversion:** Add the $9.99 offer, live Dodo checkout, idempotent webhook handling, 30-day entitlement, and seven-calendar-day refund-window representation.
4. **Production proof:** Deploy to Cloudflare; verify Convex records for user, first-use event, analysis, stage runs, payment, and access; pass empty input, malformed schedule, refresh, back navigation, duplicate webhook, and agent-timeout tests.

## 4. Features explicitly refused

- Dashboard.
- Habit tracker, streaks, recurring routines, or ongoing time tracking.
- Medical, psychological, chronotype, Dosha, personality, or productivity diagnosis.
- Generic chatbot or open-ended coaching.
- Multiple goals, multiple leaks, multiple repairs, or optimization menus in one analysis.
- Repairs that take time from sleep, essential work, caregiving, meals, or intentional rest.
- Inferred phone use, chores, relationships, commute, health, or fatigue.
- Public/private schedule details or priorities on share cards.
- Subscriptions, tiers, or pricing beyond the locked one-time $9.99 30-day pass.
- End-user Wispr Flow integration.
- LinkUp search unless the chosen priority genuinely requires one current starting resource; private schedules are never sent to it.
- ElevenLabs audio before the core first-value and checkout path works.

## 5. Exact acceptance test: first value under two minutes

**Clock:** Start when a cold user activates the primary CTA. Stop when the approved repair is fully visible and its Copy and Download controls are enabled. Checkout time is excluded.

**Fixture:** A never-before-seen email; valid sleep and wake times; one pasted schedule that accounts for all 24 hours with no material contradiction; one postponed life priority. No demo autofill and no operator assistance.

**Procedure:**
1. Open the live Cloudflare URL in a clean browser session.
2. Start a monotonic timer on primary-CTA activation.
3. Enter the four required inputs only and submit.
4. Assert that no follow-up question appears for the complete, non-contradictory fixture.
5. Wait for the final approved result.
6. Stop the timer when the repair and enabled Copy/Download controls are visible.

**Pass only if all are true:**
- Elapsed time is less than 120.000 seconds.
- Exactly one postponed priority, one defensible leak, and one conflict-free repair are shown.
- The normalized map covers exactly 1,440 minutes; overlaps are zero; unexplained gaps are zero for this fixture.
- The repair does not reduce sleep, essential work, caregiving, meals, or intentional rest and does not overlap another event.
- Every claim is grounded in supplied input; no forbidden inference or diagnosis appears.
- The response validates against `contracts/timeleak-analysis.schema.json` with no undocumented fields.
- Copy and Download both work.
- Convex contains the user, first-use event, analysis input/output, four stage-run statuses/latencies/errors, and no payment before checkout.
- The $9.99 30-day-pass offer appears only after the free repair, with immediate access and the seven-calendar-day full-refund window stated.

**Failure rule:** Any criterion failure is a failed first-value test, even if the timer is under two minutes.

## Risks and open decisions before implementation

- The repository currently contains only `AGENTS.md`; the referenced output schema is absent.
- For ruthless V1 scope, pasted/manual schedule input should be implemented first. Screenshot and voice can follow only if required for the Revenue-track demo and do not threaten the two-minute path.
- The exact Dodo refund operation and webhook event set must be confirmed against the live account/API before payment implementation.
